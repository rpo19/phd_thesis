In this section we describe advanced \acfp{ui} for \acl{ia}, developed as part of this \phd, which can be paired with an implementation of the proposed \ac{dia} to support the use cases considered in this thesis (introduced in~\Cref{sec:intro}), and we indicate which of the use cases they address.


An important contribution in this line is \acf{dave}, which has been recently
published~\cite{Agazzi2025DAVE} and released with Apache-2.0
license\footnote{\url{https://github.com/unimib-datAI/DAVE}}. It consists of an
entity-centric framework for assisting users in analyzing documents from
knowledge-intensive domains and its main \acp{ui} functionalities follow below. Furthermore, these are illustrated at the end of this section, in \Cref{fig:dave-explorer-types,fig:dave-explorer-instance,fig:dave-facets,fig:dave-qa,fig:dave-editing}, using court documents from proceedings concerning the 1995 Oklahoma City bombing.\footnote{\url{https://en.wikipedia.org/wiki/Oklahoma_City_bombing}}

\begin{itemize}
    \item  \emph{Faceted search}~\cite{ARMENTANO2014_nlp_faceted,hearst_2006_faceted}: users can retrieve documents (in support of the retrieval use cases, ref{uc:case-retrieval} and \ref{uc:investigative-retrieval}) by entering keywords, as shown in \Cref{fig:dave-facets}. Additionally, they can refine their search results by applying multiple filters corresponding to the entities in the documents or their entity types. Users can filter the corpus based on specific entities, such as individuals or locations. For example, a user interested in bombing cases in Austin, Texas might search for the keyword ``bombing'' and later narrow the focus using the filtering panel and selecting \textsc{Austin, Texas} from the list of identified entities. In this case, the resulting document will match the keyword ``bombing'' and contain a link to the entity \textsc{Austin, Texas}.

    
    \item  \emph{Document explorer}: users can visualize documents enriched with \acl{sta} by coloring the entity mentions~\cite{swoopes2025interfacedesignsupportlegal,ziwei_abstractexplorer_2025,raymond_skimming_sci_2023} according to their types (\Cref{fig:dave-explorer-types}). Each annotation is clickable allowing to inspect its details (\Cref{fig:dave-explorer-instance}).
    Furthermore, this \ac{ui} enables entity-based document exploration by providing, in a panel on the left, the list of mentioned entities grouped by type. Users can expand a type, search for specific entity, and visualize the list of all its mentions. By clicking a mention, the document explorer focuses it in the document allowing the user to see it in its entire context.  
    This interface supports the navigation and exploration use case (\ref{uc:navigation}) and implements \emph{traceability and verifiability} to a considerable extent, although the entity view is per-document rather than global.

    \item  \emph{Conversational \acl{qa}} supports natural language queries (use case \ref{uc:qa}), using the \ac{rag} paradigm, and allowing users to ask questions about entities and factual information across a single document, a selection of documents previously filtered with faceted search, or all the documents. Besides giving an answer in natural language, it provides the text passages used by the \ac{llm} for generating the answer---preserving traceability. It is illustrated in \Cref{fig:dave-qa}

    \item  \emph{Error correction and refinement}: users can refine annotations by deleting them from the document explorer, or modify the linked entity. Additionally, entity clusters can be refined with drag-and-drop by selecting two clusters from a dedicated \ac{ui} and dragging the mentions from one to another, as shown in \Cref{fig:dave-editing}. These capabilities support error correction.
    
\end{itemize}

Overall, \ac{dave} provides a unified environment that integrates entity-based document management, faceted search, conversational \ac{qa} with \ac{rag}, and \acl{hitl} refinement in a single framework. Rather than proposing new algorithms for extraction or retrieval, the system operationalizes known components in a way that is directly usable by professionals, while preserving traceability, verifiability, and error correction capability.

Finally, the graph-based visualization for \acl{ima} in investigative contexts~\cite{pozzi_wise_2025} is worth a mention: it represents communication networks derived from chats and---thanks to the underlying graph database---supports investigative queries that cannot be expressed through keyword search alone. This \ac{ui}, illustrated in \Cref{fig:kg-exploration}, was reported as useful in practitioner feedback, and its integration with the other functionalities in \ac{dave} is a direction for future work.

\medskip
\noindent
Summarizing, \Cref{tab:uc:ui} reports the coverage of the \acp{ui} presented in this section with respect to the target use cases of this thesis. While a substantial subset of the intended functionality is already supported, the coverage is not complete and exposes gaps to fill in a full implementation of the \ac{dia}. In particular:

\begin{enumerate}
    \item Lack of a deployed interface for the computation of advanced statistics (\ref{uc:stats}).
    \item Partial error correction: some of the designed correction functionalities are not available in the current implementations; for instance, entity or document deletion is not available from the \acp{ui}.
\end{enumerate}
