\section{User Interfaces and Supported Use Cases}
\label{sec:iaui-from-prototypes}

The prototypes developed in the previously published works do not only validate the architectural assumptions, but also demonstrate how the \ac{dia} can concretely support real user-facing tasks. In what follows we highlight the \acp{ui} that were actually implemented in the three prototypes and position them with respect to the five use cases introduced in~\Cref{sec:intro}. Most of the \acp{ui} originally applied to in-domain work~\cite{BELLANDI2024,pozzi_wise_2025} were later consolidated in \acf{dave}~\cite{Agazzi2025DAVE}.

\paragraph{Document explorer.}
First introduced in~\textcite{BELLANDI2024} and later continued in \acs{dave} with \textcite{pozzi_wise_2025,Agazzi2025DAVE}, the document explorer allows users to visualize annotated documents, select annotation sets, inspect properties, and navigate documents via entities. In particular, users can expand the list of entities occurring in a document, preview all mentions, and jump to a specific passage by selecting a mention. This interface supports the navigation and exploration use case (\ref{uc:navigation}) and implements \emph{traceability and verifiability} to a considerable extent, although the entity view is per-document rather than global.
Notably, this interface has been used for semi-automatic and \acl{hitl} annotation of both Italian civil judgments and investigative \ac{ima} data, as described in~\Cref{sec:judgements_labeled_data,sec:wise_dataset}.

\paragraph{Faceted search.}
\textcite{pozzi_wise_2025} introduced a faceted-search interface for investigative retrieval (\ref{uc:investigative-retrieval}) over chat documents. That same work publicly described a first version of \acs{daveold}---initially standing for \acl{daveold}---providing document exploration with error correction and faceted search. This interaction paradigm was later reused for publicly available judgments in~\cite{Agazzi2025DAVE}, simulating the case retrieval use case (\ref{uc:case-retrieval}), showing cross-domain portability.

\paragraph{Question answering and conversational interaction.}
With~\textcite{Agazzi2025DAVE}, \acs{dave} was extended with a conversational \acl{qa} interface (\ref{uc:qa}) and has been renamed to \acl{dave}. The interface supports natural language queries over a collection of documents and exposes the text passages used for answer generation to preserve traceability. At the end of this section, the \ac{dave} interfaces are shown in~\Cref{fig:dave-explorer-types,fig:dave-explorer-instance,fig:dave-facets,fig:dave-qa,fig:dave-editing}, illustrated using court documents from proceedings concerning the 1995 Oklahoma City bombing.\footnote{\url{https://en.wikipedia.org/wiki/Oklahoma_City_bombing}}



\paragraph{Graph-based exploration.}
\textcite{pozzi_wise_2025} also included a graph-based interface for investigative analysis of communication networks derived from chats, supporting complex instances of \ref{uc:investigative-retrieval} that cannot be expressed through keyword search alone. \Cref{fig:kg-exploration} shows this \ac{ui}.

\paragraph{Error correction and refinement.}
All three works provided partial support for human correction via the document explorer. In~\cite{Agazzi2025DAVE} this capability was extended with an interface for refining entity clusters---e.g.\ merge or split operations---exposing through the \ac{ui} \acl{api} already implemented in the entity registry proposed in~\textcite{BELLANDI2024}.

\paragraph{Analytical inspection.}
None of the works provided a dedicated analytical \ac{ui} for \ref{uc:stats}. \textcite{BELLANDI2024} used instead a notebook-based one.\footnote{\url{https://jupyter.org/}}

\medskip
\noindent
Summarizing, Table~\ref{tab:iaui-published} shows that the \acp{ui} prototyped so far cover a substantial subset of the target use cases, and in some cases demonstrate successful cross-domain reuse (e.g.\ faceted search). At the same time, the current coverage is not complete and reveals significant gaps that must be addressed for a fully instantiated \ac{dia}. The following gaps remain:

\begin{enumerate}
    \item \textbf{Lack of a deployed \ref{uc:stats} interface}: statistical analysis is currently supported only through notebook-based workflows and not through a dedicated, user-friendly \ac{ui}.
    \item \textbf{Partial error correction}: some of the designed error correction functionalities are still missing in the implementations (see \Cref{sec:arch_proto}).
    % \item \textbf{Limited reuse of \acp{ui} across domains}: except for the faceted search (reused in both \cite{pozzi_wise_2025} and \cite{Agazzi2025DAVE}), most implemented \acp{ui} were tailored to specific datasets or tasks rather than conceived as reusable architectural components.
\end{enumerate}

These missing capabilities directly motivated the architectural requirements for visualization and error-correction interfaces defined earlier in this chapter.
